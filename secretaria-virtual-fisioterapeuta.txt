SECRETÁRIA VIRTUAL — FISIOTERAPEUTA
Arquitetura, Fluxos e Mapa do Sistema
================================================================================


1. VISÃO GERAL — COMPONENTES
================================================================================

O sistema roda inteiramente dentro de um Docker Compose com os seguintes
serviços principais:

  - Evolution API (Baileys) — porta 8080
    Responsável pela conexão com o WhatsApp. Recebe mensagens do lead e
    expõe endpoint para o backend enviar respostas.

  - Backend NestJS — porta 3000
    Núcleo da aplicação. Recebe webhooks da Evolution API, processa as
    mensagens, aciona a IA (Claude), persiste dados e emite eventos via
    WebSocket para o frontend.

    Módulos do Backend:
      * EvolutionModule
      * LeadsModule
      * ConversationsModule
      * AiModule (Claude)
      * SchedulerModule
      * AuthModule

  - PostgreSQL — porta 5432
    Banco de dados relacional para persistência de leads, conversas,
    mensagens e agendamentos.

  - Redis — porta 6379
    Cache e fila de mensagens.

  - Frontend React (Kanban) — porta 5173
    Painel visual para o operador acompanhar e gerenciar os leads em
    tempo real via WebSocket.

  - Claude API (Anthropic) — SERVIÇO EXTERNO
    API de IA chamada pelo AiModule para qualificar leads e gerar respostas.

2. FLUXO DE UMA MENSAGEM (PONTA A PONTA)
================================================================================

  1. Lead envia mensagem via WhatsApp: "Oi, tenho dor no joelho"

  2. Evolution API recebe a mensagem e dispara POST /webhooks/evolution
     para o Backend NestJS.

  3. Backend executa:
     a. Valida a API key
     b. Extrai telefone e texto
     c. Localiza ou cria o Lead no banco (por telefone)
     d. Salva a mensagem como "inbound"
     e. Chama processMessage() → Claude API

  4. Claude API retorna JSON com:
     { reply, stage, temperature, fields }

  5. Backend:
     a. Atualiza os dados do Lead
     b. Emite evento via WebSocket para o Frontend
     c. Salva a resposta como mensagem "outbound"
     d. Chama sendTextMessage() na Evolution API

  6. Evolution API entrega a resposta ao WhatsApp do Lead:
     "Olá! Sou a Sofia, me conta mais..."

3. FLUXO DE QUALIFICAÇÃO DA IA — MÁQUINA DE ESTADOS
================================================================================

  NOVO LEAD  (primeiro contato — IA envia saudação)
      |
      v
  QUALIFICANDO  (IA realiza perguntas em 4 etapas sequenciais)
      |
      |-- Etapa 1: Sintomas       "O que você sente?"
      |-- Etapa 2: Urgência       "Há quanto tempo?"
      |-- Etapa 3: Disponibilidade "Quais horários?"
      |-- Etapa 4: Fechamento      "Valor + agenda"
      |
      |
      +---> QUENTE (score >= 70) -----> AGENDADO -----> CONVERTIDO
      |                                             \--> PERDIDO
      |
      +---> MORNO (score 40-69) -----> PERDIDO (lead desistiu)
      |
      +---> FRIO  (score < 40)  -----> LEAD FRIO (nurturing futuro)

4. KANBAN — COLUNAS E REGRAS
================================================================================

  COLUNAS:
  | NOVO LEAD | QUALIFICANDO | LEAD QUENTE | LEAD FRIO | AGENDADO | CONVERTIDO | PERDIDO |

  RESPONSÁVEL PELA MOVIMENTAÇÃO:
    - Novo Lead    → Sistema (automático na chegada)
    - Qualificando → IA
    - Lead Quente  → IA
    - Lead Frio    → IA
    - Agendado     → IA ou Operador
    - Convertido   → Operador
    - Perdido      → Operador ou IA

  O operador pode mover qualquer card via drag-and-drop em qualquer coluna.

6. BANCO DE DADOS — DIAGRAMA DE ENTIDADES
================================================================================

  Tabela: leads
    id                UUID (PK)
    phone             VARCHAR UNIQUE      ex: "5511999999999"
    name              VARCHAR
    stage             VARCHAR             valores: novo_lead | qualificando |
    temperature       VARCHAR             lead_quente | lead_frio | agendado |
    qualification_score  INTEGER          convertido | perdido
    symptoms          TEXT
    urgency           VARCHAR             valores: alta | media | baixa
    availability      VARCHAR
    budget            VARCHAR
    qualification_step INTEGER            0 a 4
    ai_context        JSONB               histórico para o Claude
    appointment_at    TIMESTAMP
    last_message_at   TIMESTAMP
    created_at        TIMESTAMP

  Tabela: conversations  (1 por lead)
    id                UUID (PK)
    lead_id           UUID (FK → leads)
    ai_enabled        BOOLEAN             operador pode desligar a IA
    created_at        TIMESTAMP

  Tabela: messages  (N por conversation)
    id                UUID (PK)
    conversation_id   UUID (FK → conversations)
    evolution_id      VARCHAR             ID da mensagem no Evolution (dedup)
    direction         VARCHAR             inbound | outbound
    sender            VARCHAR             'ai' | 'operator' | phone
    content           TEXT
    message_type      VARCHAR             text | audio | image
    created_at        TIMESTAMP

  Tabela: lead_stage_history  (auditoria de movimentações)
    id                UUID (PK)
    lead_id           UUID (FK → leads)
    from_stage        VARCHAR
    to_stage          VARCHAR
    changed_by        VARCHAR             'ai' | 'operator' | 'system'
    created_at        TIMESTAMP

  Tabela: appointments
    id                UUID (PK)
    lead_id           UUID (FK → leads)
    scheduled_at      TIMESTAMP
    duration_min      INTEGER
    service_type      VARCHAR
    status            VARCHAR             scheduled | confirmed |
    notes             TEXT                completed | cancelled
    created_at        TIMESTAMP

11. FLUXO DE NUTRIÇÃO — LEAD FRIO
================================================================================

Lead entra neste fluxo quando termina a qualificação com score < 40.
O SchedulerModule agenda automaticamente uma sequência de 4 toques.

  SEQUÊNCIA DE NUTRIÇÃO:

    D+3   Toque de empatia
          "Ei, tudo bem? Sei que é difícil conviver com dor.
           Se quiser conversar sobre opções, estou aqui 😊"

    D+7   Conteúdo educativo
          "Você sabia que dor crônica no joelho pode piorar sem
           tratamento? A fisioterapia age antes de virar cirurgia."

    D+15  Prova social / depoimento
          "A Maria, que tinha dor parecida com a sua, fez 8 sessões
           e voltou a caminhar sem dor. Posso te contar mais?"

    D+30  Oferta com urgência leve
          "Temos uma avaliação gratuita essa semana.
           Quer aproveitar? São só 30 minutos."


  DECISÃO APÓS CADA TOQUE:

    Lead responde / reabre conversa
      → Estado: REQUALIFICANDO
      → IA retoma o fluxo de qualificação do zero
      → Recalcula temperatura (re-score)
      → Segue fluxo normal: qualificando → quente → agendado

    Lead permanece em silêncio após D+60 (última tentativa)
      → "Vou deixar de incomodar, mas se mudar de ideia é só me chamar."
      → Estado: ARQUIVADO
      → Nutrição encerrada; lead fica no banco para campanhas futuras


  REGRAS DO SCHEDULER:

    Horário de envio              08h–18h, dias úteis
    Máximo de toques              4 mensagens na sequência
    Parar imediatamente se        lead responde qualquer mensagem
    Parar imediatamente se        lead pede para não receber mais
    Re-score ao reengajar         temperatura recalculada ao responder
    Operador pode pausar          toggle "pausar nutrição" no modal do lead


  CAMPOS NOVOS NA TABELA leads:

    nurture_step      INTEGER     toque atual (0 a 4)
    nurture_paused    BOOLEAN     operador pausou a sequência?
    next_nurture_at   TIMESTAMP   data/hora do próximo envio agendado


  JSON DA IA PARA MENSAGEM DE NUTRIÇÃO:

    {
      "nurture_message": "Ei, tudo bem? Sei que conviver com dor no dia
                          a dia é cansativo. Se em algum momento quiser
                          entender melhor suas opções, é só me chamar.",
      "nurture_step": 1,
      "next_step_days": 7,
      "end_sequence": false
    }

12. ORDEM DE CONSTRUÇÃO
================================================================================

  FASE 1 — Infraestrutura
    - docker-compose.yml
    - .env com todas as variáveis
    - Subir postgres + redis + evolution
    - Escanear QR Code → WhatsApp conectado

  FASE 2 — Backend Core
    - Scaffold NestJS
    - TypeORM + migrations (criar tabelas)
    - EvolutionModule: receber webhook
    - LeadsModule: criar/buscar lead por telefone
    - Eco bot: recebe mensagem → responde "recebi: {msg}"

  FASE 3 — IA
    - AiModule com Claude API
    - System prompt da Sofia
    - Parsing do JSON de resposta
    - Integrar na pipeline (substituir eco bot)
    - Testar fluxo completo de qualificação

  FASE 4 — Frontend
    - Scaffold React + Vite + shadcn/ui
    - LeadsService (chamadas REST)
    - KanbanBoard com dados estáticos
    - useWebSocket hook (Socket.io)
    - Cards se movem em tempo real
    - LeadDetailModal com conversa

  FASE 5 — Polish
    - Página de QR Code (para parear WhatsApp)
    - Stats no header (contadores por coluna)
    - Toggle IA por lead
    - Envio manual pelo operador
    - Histórico de movimentações de stage
